# Contributing to Somniascope Widget

Thank you for your interest in contributing to the Somniascope Widget project! We welcome contributions from everyone. Please follow the guidelines below to ensure a smooth contribution process.

## Code of Conduct
We expect all contributors to adhere to our [Code of Conduct](link_to_code_of_conduct). Be respectful, inclusive, and considerate of others in the community.

## Bug Reporting
If you encounter a bug, please report it by following these steps:
1. Search the issue tracker to see if the bug has already been reported.
2. If not, create a new issue and provide a clear and descriptive title along with the steps to reproduce the bug. Attach any relevant screenshots or error messages.

## Pull Request Process
1. Fork the repository and create your branch from `main`.
2. Make your changes and ensure they follow the guidelines below.
3. Commit your changes with a descriptive message.
4. Push to your fork and submit a pull request. Ensure your PR description is detailed.

## Development Setup
To set up your development environment:
1. Clone the repository: `git clone https://github.com/bryanssss/somniascope-widget.git`
2. Install the required dependencies: 
   ```
   npm install
   ```
3. Create a `.env` file for environment variables as specified in `.env.example`.

## Code Style Guidelines
Please adhere to the following coding standards:
- Follow [StandardJS](https://standardjs.com/) for JavaScript files.
- Ensure proper indentation and meaningful variable names.

## Commit Message Guidelines
Your commit messages should be clear and meaningful. Use the following structure:
- Use the present tense. Example: "Fix bug in login button"
- Include an issue number if applicable: "Fix #123"

## Documentation Requirements
Any new feature should include documentation. Update the README or relevant documentation files as necessary to reflect your changes.

## Testing Guidelines
Make sure to add tests for any new features or bug fixes. Run the test suite before submitting your pull request to ensure all tests pass.

## Review Process
Once your pull request is submitted, it will be reviewed by one of the maintainers. Address any feedback and make necessary changes. Once approved, your pull request will be merged into the main branch.

Thanks for contributing! We appreciate your help in making Somniascope Widget better!
